Phase 1
Metadata Migration
2–3 weeks  ·  MongoDB → NextGen MongoDB
1
Extract Collections
Batch cursor reads with resume-token checkpointing for safe replay
2
Transform + Schema Map
Field renaming, type coercion, siteId assignment per spec document
3
Load → NextGen MongoDB
Upsert (idempotent): users · rooms · subscriptions · members
4
Thread Derivation Pre-pass
MongoDB $group aggregation with allowDiskUse:true
Synthesizes threadRooms + threadSubscriptions — these collections do not exist in tchat 2.0. Required before Phase 3 can run.
5
Metadata Reconciliation
Count assertions per collection before marking phase complete
➔
Phase 2
Cross-Site Sync
1 week  ·  Federation of metadata across sites
1
Remote User Shadows
Copy user records to every site where they are a room member
2
Room Metadata Copies
Rooms with cross-site members replicated to member home sites
3
Subscription Copies
siteId = room's site, not user's home site — must be split across sites correctly
4
Cross-Site Thread Subscriptions
Foreign thread repliers → ThreadSubscription written to their home site
Hard dependency for Phase 3: the message transformer looks up threadRoomID from threadRooms collection. That lookup only succeeds if Phase 1 + Phase 2 are fully complete.
➔
Phase 3
Message Migration
3–4 weeks  ·  75M+ messages  ·  Dual-sink writes
1
Extract Messages
Per-room batch cursor with resume-token checkpoint (50 rooms concurrently via Airflow pool)
2
Transform + Map
Schema mapping · Cassandra time-bucket computation · threadRoomID lookup
3
Write → Cassandra
Bulk writes: messages_by_room + thread_messages_by_thread (LWW idempotent)
4
Index → Elasticsearch
Bulk API · external version = createdAt_ms for safe re-index
5
Advance Checkpoint
Only after both sinks confirm. Failed records → Quarantine (not silent drop)
Alert fires if quarantine rate exceeds 1% of batch size

Reconciliation Gate  ·  All checks must pass before cutover

✓
Count Verification
Per-room: source message count == Cassandra partition count
✓
1% Content Sampling
Random 1% of message IDs fetched from target — field-level equality check
✓
Thread Integrity Check
Distinct threadParentMessageIDs in source == threadRooms count in target
✓
Quarantine Rate Check
Total quarantined records must be below 1% threshold before proceeding

Cutover Sequence  ·  Transition to Live Migration

→
All Reconciliation Checks Pass
Gate clears across all sites simultaneously
→
CDC Go Connector Activated
Resumes from T₀ resume token captured before batch migration began
→
Live Traffic Switched Over
T₀ token fills the delta gap between batch completion and cutover moment
→
Drain Quarantine Store
Manual review and replay of quarantined records post-cutover
