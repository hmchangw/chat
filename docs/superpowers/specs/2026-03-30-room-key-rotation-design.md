# Room Key Rotation — Design Spec

**Date:** 2026-03-30
**Status:** Approved
**Scope:** `pkg/roomkeystore` (modify existing package)

## Overview

Extend `pkg/roomkeystore` with two related changes:

1. **TTL removal** — the current key for a room no longer expires automatically. Keys persist until explicitly deleted or rotated.
2. **Key rotation** — a new `Rotate` method swaps in a fresh key pair, moves the outgoing key to a "previous" slot with a configurable grace-period TTL so messages encrypted under the old key remain decryptable for that window.

## Motivation

An always-expiring current key creates operational risk: if the key expires before a replacement is set, the room becomes un-encryptable. Explicit rotation gives services full control over when keys change. The grace period ensures messages encrypted under the old key can still be decrypted during a rolling deployment or reprocessing window.

## Architecture

This is an in-place evolution of `pkg/roomkeystore` — no new packages or files. Changes land in `roomkeystore.go`, `roomkeystore_test.go`, and `integration_test.go`.

### Valkey Layout

Two hashes per room, each with a distinct lifecycle:

| Valkey key | TTL | Purpose |
|---|---|---|
| `room:{roomID}:key` | none | Current key pair (active for encryption) |
| `room:{roomID}:key:prev` | `GracePeriod` | Previous key pair (readable during grace window) |

The previous hash is created only by `Rotate` and auto-expires when the grace period elapses.

## Data Model

### New type: `VersionedKeyPair`

```go
// VersionedKeyPair pairs a key pair with its caller-assigned version identifier.
type VersionedKeyPair struct {
    VersionID string
    KeyPair   RoomKeyPair
}
```

`VersionID` is supplied by the caller (typically a UUID generated by the service writing the key). The library never generates version IDs — that responsibility belongs to the consuming service.

### Hash fields

Both hashes store three fields:

| Field | Value | Encoding |
|---|---|---|
| `pub` | 65-byte P-256 uncompressed public key | standard base64 |
| `priv` | 32-byte P-256 private key scalar | standard base64 |
| `ver` | caller-assigned version ID string | raw string |

## Interface & Config

### Updated `RoomKeyStore` interface

```go
type RoomKeyStore interface {
    Set(ctx context.Context, roomID string, pair RoomKeyPair) error
    Get(ctx context.Context, roomID string) (*VersionedKeyPair, error)
    GetByVersion(ctx context.Context, roomID, versionID string) (*RoomKeyPair, error)
    Rotate(ctx context.Context, roomID string, newPair RoomKeyPair) error
    Delete(ctx context.Context, roomID string) error
}
```

### Updated `Config`

```go
type Config struct {
    Addr        string        `env:"VALKEY_ADDR,required"`
    Password    string        `env:"VALKEY_PASSWORD" envDefault:""`
    GracePeriod time.Duration `env:"VALKEY_KEY_GRACE_PERIOD,required"`
}
```

`KeyTTL` is removed. `GracePeriod` replaces it, applying only to the previous key after a rotation.

### Behaviour contracts

- **`Set`** — writes `pub`, `priv`, `ver` to `room:{roomID}:key` with no TTL. Overwrites any existing current key. Does not touch the previous key slot.
- **`Get`** — returns `(*VersionedKeyPair, nil)` for the current key. Returns `(nil, nil)` if no current key exists.
- **`GetByVersion`** — reads both the current and previous hashes. Returns `*RoomKeyPair` for whichever hash has a matching `ver` field. Returns `(nil, nil)` if neither matches or both hashes are absent.
- **`Rotate`** — pipelines two operations atomically: (1) copy the current hash fields to `room:{roomID}:key:prev` and set `EXPIRE = GracePeriod`; (2) write the new pair to `room:{roomID}:key` with no TTL. Returns `ErrNoCurrentKey` if no current key exists. Replaces any existing previous key.
- **`Delete`** — removes both `room:{roomID}:key` and `room:{roomID}:key:prev` in a single pipeline. No-op if either or both are absent.

## Error Handling

- All errors wrapped with context: `fmt.Errorf("rotate room key: %w", err)`, `fmt.Errorf("get room key by version: %w", err)`, etc.
- **`ErrNoCurrentKey`** — exported sentinel error returned by `Rotate` when the current key slot is empty. Callers use `errors.Is(err, ErrNoCurrentKey)` to detect this and generate a fresh key pair instead.
- `GetByVersion` returning `(nil, nil)` is not an error — it means the version is unknown or the previous key has expired. Callers must handle this as a cache miss.

## Implementation Notes

### `hashCommander` interface extension

Add one new method to the internal `hashCommander` interface to support the atomic rotate pipeline:

```go
rotatePipeline(ctx context.Context,
    currentKey, prevKey string,
    pub, priv, ver string,
    gracePeriod time.Duration,
) error
```

This single method encapsulates the pipeline so the unit test fake can simulate it without needing to understand pipeline internals. The `redisAdapter` implementation executes a `redis.Pipeline` with `HSet` + `HSet` + `Expire`.

`deletePipeline` replaces the existing `del` method to atomically delete both keys:

```go
deletePipeline(ctx context.Context, currentKey, prevKey string) error
```

### `Set` change

Remove the `expire` call. `Set` calls only `hset` — no TTL is applied.

### `Get` change

`Get` now reads the `ver` field alongside `pub` and `priv` and returns a `*VersionedKeyPair`.

## Configuration

| Env var | Required | Description |
|---|---|---|
| `VALKEY_ADDR` | yes | `host:port` of the Valkey instance |
| `VALKEY_PASSWORD` | no | Auth password (empty = no auth) |
| `VALKEY_KEY_GRACE_PERIOD` | yes | Duration the previous key remains readable after rotation (e.g. `24h`, `168h`) |

`VALKEY_KEY_TTL` is removed.

## Testing

### Unit tests (`roomkeystore_test.go`)

Extend existing table-driven tests. New scenarios:

| Scenario | Method |
|---|---|
| `Get` returns `VersionedKeyPair` with correct `VersionID` | `Set` + `Get` |
| `GetByVersion` matches current key | `GetByVersion` |
| `GetByVersion` matches previous key | `GetByVersion` |
| `GetByVersion` returns `(nil, nil)` — no match | `GetByVersion` |
| `GetByVersion` Valkey error | `GetByVersion` |
| Rotate happy path — new pair becomes current, old becomes previous | `Rotate` |
| Rotate with no current key returns `ErrNoCurrentKey` | `Rotate` |
| Rotate replaces existing previous key | `Rotate` |
| Rotate pipeline Valkey error | `Rotate` |
| `Delete` removes both current and previous | `Delete` |
| `Delete` no-op when both absent | `Delete` |

Existing `Set`/`Get`/`Delete` tests are updated to reflect the new signatures (`Get` now returns `*VersionedKeyPair`; `Delete` now removes both hashes).

The `fakeHashClient` gains `rotatePipeline` and `deletePipeline` methods with injectable errors (`rotatePipelineErr`, `deletePipelineErr`).

### Integration tests (`integration_test.go`)

New tests:

- **Rotate round-trip** — `Set` initial pair, `Rotate` to new pair, verify `Get` returns new pair, verify `GetByVersion` returns old pair by its version ID, verify `GetByVersion` returns new pair by its version ID.
- **Grace period expiry** — rotate with a 1-second `GracePeriod`, verify old key retrievable immediately after, sleep 2s, verify `GetByVersion` returns `(nil, nil)` for old version.
- **Rotate with no current key** — verify `errors.Is(err, ErrNoCurrentKey)`.
